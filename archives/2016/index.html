nodejs

  • exports和module.exports的区别

    • module.exports 初始值为一个空对象 {}
    • exports 是指向的 module.exports 的引用
    • require() 返回的是 module.exports 而不是 exports

  • npm需要注意的：

    • -S ：等价于 –save

    • npm i 等价于 nam install

    • npm i express –save –save-exact，可以将版本号写入dependencies，推荐写法；

      运行以下命令，在以后的安装中都会锁定版本号，相当于加了–save-exact；
      npm config set save-exact true。（npm config set 命令将配置写到了 ~/.npmrc 文件，运行 npm config list 查看。）

    • npm script 提供了强大的缩写命令功能，如npm start 等价于 npm run start；npm test 等价于 npm run test

    • 运行npm shrinkwrap会生成npm-shrinkwrap.json，将版本锁定，则运行 npm install 的时候会优先使用 npm-shrinkwrap.json 进行安装，没有则使用 package.json 进行安装。ps：如果node_modules下存在的某个模块在package.json中没有，运行npm shrinkwrap会报错，另外，npm shrinkwrap 只会生成 dependencies 的依赖，不会生成 devDependencies 的。

    • supervisor和nodemon，supervisor和nodemon 会监听当前目录下 node 和 js 后缀的文件，当这些文件发生改动时，supervisor 会自动重启程序；如supervisor –harmony index, nodemon —exec babel-node ./

    先启一个简单node服务：

    req 包含了请求来的相关信息，常用的req属性：

    • req.query: 解析后的 url 中的 querystring，如 ?name=haha，req.query 的值为 {name: 'haha'}
  • req.params: 解析 url 中的占位符，如 /:name，访问 /haha，req.params 的值为 {name: 'haha'}

  • req.body: 解析后请求体，需使用相关的模块，如 [body-parser](https://www.npmjs.com/package/body-parser)，请求体为{“name”: “haha”}，则 req.body 为{name: ‘haha’}`
