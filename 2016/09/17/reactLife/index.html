理解react组件的生命周期

17 Sep 2016 en react


对react生命周期的重新理解：

一个组件的声明周期可以理解为三个阶段：

  • 初始化阶段：mounting 组件正在插入到 DOM 中
  • 运行中状态：updating 组件正在重新注入新数据后更新到 DOM 中
  • 销毁阶段：unmounting 组件从 DOM 中移除

mounting 阶段

执行顺序：getDefaultProps => getInitialState => componentWillMount => render => componentDidMount

  • getInitialState() 首先被被调用，返回原始数据，默认的数据。

  • componentWillMount() 组件即将被装载、渲染到页面上（render之前最好一次修改状态的机会.

  • componentDidMount() 在组件 mounting 完成后调用，

updating 阶段

componentWillReceiveProps => shouldComponentUpdate => componentWillUpdate => render => componentDidUpdate

  • componentWillReceiveProps(nextProps) 在接收到新的 props 时调用
    nextProps: 表示所有props对象，

  • shouldComponentUpdate(nextProps, nextState) 组件接受到新属性或者新状态的时候,在组件需要更新 DOM 时调用，若这个函数返回 false 则告诉 React 不要更新, 如定时器到多少停止，或者倒计时组件等。

  • componentWillUpdate(nextProps, nextState) 不能修改属性和状态

  • componentDidUpdate(prevProps, prevState) 在这里可以修改DOM

unmounting 阶段

  • componentWillUnmount() 在组件移除时被调用，在这里可以对数据进行清理，譬如清除定时器，绑定的事件等。
    如：React.unmountComponentAtNode(document.getElementById("container"));

强制使用数据更新组件

  • forceUpdate() 强制使用数据更新组件，而不用调用 this.setState()

浏览器的执行顺序

render时自动执行的顺序:
getInitialState() –> componentWillMount() –> render() –> componentDidMount() –> componentUnMount
更新state时的执行的函数:
componentWillReceiveProps -> shouldComponentUpdate (return true继续执行，如果return false停止) -> componentWillUpdate -> render() -> componentDidUpdate()
