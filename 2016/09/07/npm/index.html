npm常用命令集合

07 Sep 2016

npm—Node Package Manager，是随同NodeJS一起安装的包管理和分发工具，它很方便让JavaScript开发者下载、安装、上传以及管理已经安装的包。
在安装nodeJs的情况下用npm安装所需要的包，很方便。

  • npm install 相应的文件夹安装
  • npm install -g 全局安装
  • npm install @ 可指定版本号安装。如：npm install grunt,或者npm install grunt@3.2.1
  • -S,–save 安装包信息将加入到dependencies（生产阶段的依赖）,如：npm install grunt –save 或者 npm install grunt –S
  • -O, –save-optional 安装包信息将加入到optionalDependencies（可选阶段的依赖）,如npm install grunt –optional 或者 npm install grunt –O
  • -E, –save-exact 精确安装指定模块版本,如：npm install grunt –save-dev 或者 npm install grunt -D

模块的依赖都被写入了package.json文件的dependencies下。

  • npm install，会安装dependencie下所有配置好的依耐包
  • npm uninstall 卸载模块，npm uninstall –save-dev卸载相应模式
  • 查看已安装的模块：npm ls或（ll,list,la），查看全局安装：npm ls -g
  • 查看模块的版本号npm version
  • 在项目中引导创建一个package.json文件 ：npm init
  • 查看包安装的路径npm root [-g]，输出node_node_modules的路径
  • npm view moudleName dependencies：查看包的依赖关系
  • 启动模块：npm start
  • 停止模块：npm stop
  • 搜索模块：npm search express
  • 更新模块: npm update express
  • 在 npm 资源库中注册用户：npm adduser;

  • 发布模块: npm publish

    package.json的语法

  • name - 包名。
  • version - 包的版本号。
  • description - 包的描述。
  • homepage - 包的官网 url 。
  • author - 包的作者姓名。
  • contributors - 包的其他贡献者姓名。
  • dependencies - 依赖包列表。如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下。
  • repository - 包代码存放的地方的类型，可以是 git 或 svn，git 可在 Github 上。
  • main - main 字段是一个模块ID，它是一个指向你程序的主要项目。就是说，如果你包的名字叫 express，然后用户安装它，然后require(“express”)。
  • keywords - 关键字
