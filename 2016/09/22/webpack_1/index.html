Webpack入门

22 Sep 2016

什么是Webpack?
模块打包的工具，主要功能：
1.分析你的项目结构，分析js模块，找到一些浏览器不能识别的语言（scss，TypeScript，CoffeeScript、ES6等），用loader加载器将整个项目打包成浏览器识别的语言和便与开发.
2.将样式文件和图片等静态资源也可视为模块进行打包。
3.支持模块加载器和插件机制。
4.支持 SourceUrls，更加易于调试。
5.可以通过配置，打包成多个文件。有效利用浏览器的缓存功能提升性能。(例如审计项目)。
6.webpack有一个文件束chunkFile的概念，它会自动去把需要异步加载的文件变成一个chunkFile，然后触发加载的时候再去加载chunkFile。

工作方式

从整个项目指定的入口文件出发（一般为index.js），从这个文件中找到项目所有依赖的文件，将他们通过loader加载器处理，打包成浏览器认识的Js文件。
什么是loader加载器？
它就是用来识别webpack加载的不同的文件，把他们解析成浏览器能识别的文件。譬如要打包sass文件，需要安装：npm install sass-loader node-sass，然后再配置文件中添加：
{ test: /\.sass$/, loaders: ["style", "css", "sass"] }。

Source map就是一个信息文件，里面储存着位置信息。也就是说，转换后的代码的每一个位置，所对应的转换前的位置。一般启用方法：在在转换后的文件中添加 //@ sourceMappingURL=jquery.min.map。Chrome浏览器支持这个功能。在Developer Tools的Setting设置中，确认选中”Enable source maps”。想要了解更多可以参考这篇文章
